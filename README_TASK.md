## 1. Задача

Разработать консольное приложение в ОС Linux на языке Python/С++ для
детектирования ТС грузового и легкового типа в режиме realtime (условно 20+ fps
на Nvidia gt 1000+ серии). Приложение принимает в виде аргументов командной
строки абсолютный путь к модели, входному и выходному видеофайлу, например:
```
python demo_detection \
    --model_path '/abs/path/to/demo/model.pt' \
    --input_data '/abs/path/to/demo/input_video.avi' \
    --output_data '/abs/path/to/demo/output_video.avi'
# или для С++ версии
./demo_detection \
    --model_path '/abs/path/to/demo/model.pt \
    --input_data '/abs/path/to/demo/input_video.avi' \
    --output_data '/abs/path/to/demo/output_video.avi'
```

На входном видео отрисовывается детекция и записывается в выходной видеофайл
Результат выполнения тестового задания необходимо запаковать в zip-архив,
который содержит:
* текстовый файл со ссылкой на git с кодом запуска,
* выходные данные с отрисованной детекцией,
* логи/графики/метрики обучения актуальной модели.

Примечание: 
* достаточно весь код (детекции и обучения) написать на Python, отдельное
  приложение для запуска на С++ необязельно, но будет плюсом,
* цвета отрисовки значения не имеют,
* учитывание других классов типа прицеп, мотоцикл и пр. значения не имеют.

## 2. Цели

Продемострировать навыки или способность разобраться в темах:
* написание Python/C++ кода и ведение рабочего процесса в git,
* способность работы с данными и последующем обучением модели,
* интерпритация процесса и результатов обчуения, валидации, тестирования.

Примечание: некорректный результат -- это тоже результат и нужно его объяснить,
предложить способы решения/улучшения

## 3. Исходники для тестового задания:

* `train_data` -- xml-файлы с разметкой + папки изображения к ним.
* `test_data` -- видеофайл и папка с изображениями, на которых нужно запустить
  приложение для демонстрации.

## 4. Предполагаемые этапы решения

* Определиться с моделью для детекции, которая может обеспечить необходимую
  производительность.
* Написать заготовку для запуска модели на данных.
* Оценить качество исходной модели (необязательно).
* Дообучить модель на исходных данных, для чего необходимо:
   - написать парсер xml-файлов,
   - из xml-данных сгенерировать обучающую выборку в виде пар "входные
     данные -- выходные данные" и запустить обучение.
* После дообучения запустить модель на видео из `test_data` для демонстрации результата.
